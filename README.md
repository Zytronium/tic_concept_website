# Oklahoma Technology & Innovation Association (OKTIA) Website

The OKTIA website, soon to be www.oktia.org, is the destination for follow-up from discussoins, membership 
solicitations, and marketing efforts. It is currently a WIP prototype, it is built with Next.js, React, TailwindCSS, 
and TypeScript.

## OKTIA Website Notes

### Things I'll need to complete the website:

- What font(s) to use. If its a paid font, I'll need the font file so I don't need to purchase it too.
- What color scheme to use (until confirmed, the prototype will be built using green, red, and white based on the logo)
- A wireframe to base the website layout on
- Precisely what text to put on the website and where, word for word (I can take some creative liberty if needed though)
- .PNG image file of the OKTIA logo (transparent background preferred but not required) (for now, I'm using an AI-generated approximation generated from a picture of the logo on the paper document I was given)
- .PNG, .SVG, or .ICO image of the website icon (square shape with a transparent background preferred) (using something I created from your logo as a placeholder for now)
- Preferably some landscape picture to put in the about/hero section

This is a non-exhaustive list and may be expanded.

### Questions
- Will both phase 1 and phase 2 need to be completed by the deadline, or just phase 1?
- Is the given deadline an exact date it must be completed by, or just a goal?

### Project Timeline
- Phase 1:
  - Get the basic website up and running, nothing complex yet, no API or interactive features
  - Include logo, mission statement, description of organization, who we are (board of directors & profiles), membership levels, how to contribute, and FAQ.
- Phase 2:
  - Add interactive features:
    - Event schedule
    - Integrated giving functionality
    - Others(?)
- Phase 3 (?):
  - Add a CMS (Content Management System) for the website, which allows the client to change most content on the website (text, images, etc.) without editing the code or structure of the website
  - Anything more complicated than changing text, images, and positioning of text and images on certain pages would still require a paid developer to edit code
  - This phase is optional but recommended and would be paid in addition to the payment for phases one and two

Preliminary landing page target date is June 15th, 2026.

Prototype hosting will be done on Vercel.
Production hosting has not been decided yet, but will likely be Google Cloud, Vercel, Netlify, Cloudflair, or others - whichever has the best combination of developer ease and price for expected traffic and usage.
